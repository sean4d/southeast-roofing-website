# SEO / AI-discoverability audit evidence — 2026-07-21

Raw artifacts from the production verification audit of `https://southeastroofing.llc`
(post PRs #74–#77). Evidence only — generated by read-only HTTP tests against production
and repo inspection. No production behavior was changed to produce these files.

**Target (canonical) host:** `https://southeastroofing.llc`
**Repo state audited:** `main` (includes #74 GBP/SEO, #75 copy-of redirects, #76 legacy 404 redirects + Claude-SearchBot, #77 sitemap canonical fix + smoke test).

## Headline results

- **Redirects:** 42/42 resolve to HTTP 200 on the canonical apex. Every first hop is permanent (308). Apex requests are 1 hop; www/roofs.ms/http variants are 2–3 hops (host-canonicalization hop, then path hop) — inherent to Vercel edge domain redirects, all landing on the correct apex page.
- **Sitemap:** 113 URLs — 113 HTTP 200, 0 redirects, 113 indexable, 113 self-canonical, exactly one H1 each, 0 missing/duplicate titles, 0 missing/duplicate descriptions.
- **Internal links:** 120/121 targets 200 (the one `000` was a transient timeout on `/commercial`, re-verified 200); 0 non-canonical-host links; 0 orphans.
- **Structured data:** 0 JSON-LD parse errors; exactly one `RoofingContractor` business entity per page with stable `@id …/#organization`.
- **Smoke test:** 29/29 pass against production.

## Reproduce

```bash
npm run seo:smoke                       # default: https://southeastroofing.llc
node scripts/seo-smoke.mjs <BASE_URL>   # test another origin
```

The `redirect-matrix.csv`, `sitemap-audit.csv`, and header dumps were produced with `curl`
(status, `%{redirect_url}`, `%{num_redirects}`, `%{url_effective}`) and standard text tooling.

## Limitations (stated for honesty)

- **Lighthouse / PageSpeed:** not captured. The keyless PageSpeed Insights API returned HTTP 429
  (daily quota exhausted for this environment's shared egress), and sandboxed Chrome cannot reach
  production. Run each URL through <https://pagespeed.web.dev> (mobile + desktop) or `npx lighthouse`
  locally for scores. Curl-measured signals were healthy (Brotli, TTFB 0.27–0.75s, 28–40 KB HTML).
- **Search Console:** the supplied exports were aggregate counts + one 404 drilldown. Individual URLs
  in "Discovered/Crawled – currently not indexed" cannot be mapped from that data; use each report's
  per-issue Export in GSC. No indexing dates are implied.
- **Structured data** was validated for JSON parse + entity/type structure, not via Google's Rich
  Results API — spot-check 2–3 URLs in the Rich Results Test for eligibility confirmation.
- The internal-link `000` timeout is a crawl-time artifact, not a site defect.
